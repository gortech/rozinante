# Zig 0.16 notes

API pitfalls and conventions this codebase already hit on Zig 0.16.0. Check here before writing `std.mem` trims, `std.Io`/`Writer` code, `@embedFile`, or anonymous-struct returns.

- **`std.mem.trimRight` → `std.mem.trimEnd`** (renamed). `Writer.flush()` no longer takes a `self` parameter. Watch for similar `std.mem` / `Io` Writer renames.
- **`@embedFile` cannot resolve paths outside the module's package root.** Data files for `src/` modules must live under `src/` (hence `src/data/openings.tsv`, not a repo-root `data/`). No `build.zig` change needed when the file is within the module root.
- **Each anonymous struct literal is a distinct type**, causing declaration-vs-return type mismatches. Use named structs (e.g. `MovetextResult`) for returns.
- **The new `std.Io.Writer` lacks `print()`.** Persistence uses a buffer-return approach (`writePgn` returns `[]const u8` from a caller buffer) instead, which also matches the project's stack-allocation conventions.
- **`std.fmt` rejects the `+` sign flag.** `"{d:+.1}"` is a comptime error ("expected . or }, found '+'"); prepend the sign manually (`if (v >= 0) "+" else ""` then `"{d:.1}"`). This only surfaces in `zig build` (the executable), not `zig build test` — a private function the tests never reach is not analyzed, so its comptime format string is never checked. **Run `zig build` as well as `zig build test`** before declaring a change done.
- **`renderer.writeStr` stores a *slice* into your `text`, not a copy** (a vaxis `Cell.grapheme` is `[]const u8`). The string must stay alive until `vx.render` flushes — and `renderGame` / `viewer.render` return *before* `vx.render` runs, so a value formatted into a render-local buffer (`std.fmt.bufPrint` into a stack array, or a by-value `SanNotation` whose `.slice()` you pass) dangles and paints **blank**. String literals and the `writeNum` digit strings are static, so they always survive — which is why this only bites dynamically-formatted values. Render numbers with `writeNum` + a literal suffix, or stash the formatted string in a frame-stable field (a `Game` / `ViewerState` field that lives across the flush), never a render-local. Offline render tests (build a `vaxis.Screen` + `Window`, render, read cells back) catch it.
